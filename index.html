Part 2
  • Use of HTML tags
  • Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag (the paragraph tags, or the bold and italics tags are examples of this). Some tags (called "void" tags) do not require a closing tag (like the "break" tag).
  • Element: An HTML element refers to everything within a set of opening and closing tags.
  • Attribute: This is a property of an HTML element. The href attribute of the "a" element, specifies a link, for example.
  • Inline vs. block tags: Inline elements are just part of the text (eg. bold, emphasis, image link, website link, break tags). Block elements make an invisible box (eg. paragraph tag, div tag).
  • Container elements:Span and div. They just "contain their content". Span is inline, whereas div is block. Styles can be assigned to them to change how the content looks like.

Exploring the web

We can use Google Chrome's developer tools to explore the structure of a website. When opening them, we can see the elements of the website and other useful information as the CSS style for the different parts. By clicking on a part of the code, we see in the browser the part of the website it refers to highlighted.

Page Structure

The browser creates a "tree-like" structure of the page.

  • HTML and CSS are both "languages"
  • HTML controls the structure and the content of a web page. It has syntax and rules.
  • CSS controls the style of a page (how it looks).
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.
  • Everything on a webpage is actually in a box (remember the circles and squares example). In fact, you "boxify" a design before creating a website.

Lesson 1.3: Adding CSS style and HTML structure

The structure of an HTML file

In any file, you need to first specify that the document is an HTML document. The line <!DOCTYPE html>, written at the beginning of an HTML document, tells the browser the type of document that it is opening.

Then, you need to add the "top level tag" <html>, because everything in the document is html. The end tag </html>has to be placed at the end of the document

In order to make our code easier to read, we indent the contents of the site.

After the html tag, you need to add the head tags (<head></head>). There, you can add a link to the style file. To do so, you can type:
<link rel="stylesheet" type="text/css" href="style.css">

After the head tag, we have to add the "body" tag (<body></body>).

Since the website includes a link to the CSS file, we can control how our website looks like from the CSS file. An alternative to using CSS separate sites is to specify the style of our site in between an opening and closing <style> tag.

CSS style

CSS helps us to control how our website looks like. It stands for "Cascading Style Sheets" and allows programmers to avoid repetitions. You can have several CSS files assigned to your website, and assign them different levels of priority/applicability.

In CSS, the most specific rule is applied to every element.

The first thing you write is a "selector". This defines what elements of the page a particular style will be applied to. You can select an html tag, and that will apply to all such tags in the page:
p { } Another option is to define styles by class:
.class { attribute:value; }

How to include CSS in a website
  1. write CSS in the <head> of your HTML. This method is good for very small projects.
  2. Link your HTML to a separate CSS file.
    This lets you stay more organized when working on larger projects.
    1. Write all of your structural HTML in one file.
    2. In a separate file, put all of the CSS code.
    3. Add a <head> element to the top of your HTML
    4. Add a <link> tag inside the <head> element (*link is void, so it does not need a closing tag!)
    5. Add the following attributes to your <link>: rel="stylesheet" type="text/css" href="yourCSSfile.css"
  3. Write your style inline with your HTML. This is generally a very bad idea because it leads to lots of repeated code.
Box sizing and positioning

There are four main points about box sizing which are important:

  1. HTML elements are boxes and each box has 4 components: content, padding, border, margin (inside-outside).
  2. Having so many components in each box makes it hard to get the size of a box right.
  3. There are two techniques you can use to help deal with sizing issues:
    (a) Set sizes in terms of percentages rather than pixels. (b) Set the box-sizing attribute to border-box for every element.
  4. Different browsers work slightly differently. Sometimes this causes that they display the same code differently.
Box positioning
  1. Divs are block elements, so by default they take up the entire width of a page.
  2. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
Code, Test, Refine
  1. Look for natural boxes. Use the "divide and conquer" strategy. Look for places that are distinct. Everything in the web is a box.
  2. Look for repeated styles and semantic elements. Think how style can be related to the different HTML elements.
  3. Write your HTML.
  4. Apply your styles. Go from the biggest to the smallest.
  5. Fix things and repeat stages 3 and 4 until your website looks like the designed one. You can use Google's developer tools to see how the site changes before you actually implement any changes.
